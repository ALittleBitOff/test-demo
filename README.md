# README для приложения по сортировке

## Описание проекта
Данное приложение разработано на языке Java и реализует алгоритмы сортировки и бинарного поиска с использованием дженериков и паттерна "Стратегия". Приложение предоставляет пользователю цикл взаимодействия с возможностью выбора способа заполнения массива, а также другим функционалом.

## Возможности приложения
- **Цикличный интерфейс** для взаимодействия с пользователем.
- **Опции для заполнения исходного массива данных**:
    - Из файла
    - Рандомно
    - Ввод вручную
- **Выбор длины массива**.
- **Реализация бинарного поиска** для поиска элементов в отсортированной коллекции.
- **Кастомные реализации классов**:
    - **Автомобиль**: содержит поля `мощность`, `модель`, `год производства`.
    - **Книга**: содержит поля `автор`, `название`, `количество страниц`.
    - **Корнеплод**: содержит поля `тип`, `вес`, `цвет`.

## Архитектура приложения

### Основные классы
| Класс       | Поля                                  |
|-------------|---------------------------------------|
| Автомобиль  | `мощность: int`, `модель: String`, `год: int` |
| Книга       | `автор: String`, `название: String`, `кол-во страниц: int` |
| Корнеплод   | `тип: String`, `вес: double`, `цвет: String` |

### Интерфейсы и классы
- **Интерфейс `Sorter<T>`**: определяет метод `sort(T[] array)`.
- **Класс `ShellSort<T>`**: реализует алгоритм сортировки Шелла для переданных дженериков.
- **Интерфейс `Searchable<T>`**: определяет метод `binarySearch(T[] array, T key)`.
- **Класс `CustomArrayList<T>`**: реализация собственного списка (ArrayList).

### Паттерн "Стратегия"
Создание разных стратегий сортировки, которые могут быть внедрены в приложение. Например, `ShellSort` для сортировки массивов.

## Как начать

1. **Клонируйте репозиторий:**

   ```bash
   git clone https://github.com/ваш-username/название-репозитория.git
   ```

2. **Перейдите в директорию проекта:**

   ```bash
   cd название-репозитория
   ```

3. **Создайте новую ветку для ваших изменений:**

   ```bash
   git checkout -b feature/ваша-ветка
   ```

4. **Внесите изменения в проект и сохраните их.**

5. **Добавьте изменения в индекс:**

   ```bash
   git add .
   ```

6. **Сделайте коммит:**

   ```bash
   git commit -m "Ваше сообщение коммита"
   ```

7. **Отправьте изменения на GitHub:**

   ```bash
   git push origin feature/ваша-ветка
   ```

## Создание пул-реквеста
1. **Перейдите на страницу вашего репозитория на GitHub.**
2. **Нажмите на кнопку "New Pull Request".**
3. **Выберите ветку, которую хотите слить (например, main), и ветку с вашими изменениями (например, feature/новая-фича).**
4. **Напишите описание изменений и нажмите "Create Pull Request".**
5. **Дождитесь ревью и одобрения вашего пул-реквеста.**
